> **Uzhavan AI** is an AI-powered farming copilot built for Indian smallholder
> farmers. Through a single unified web dashboard, it helps a farmer pick the
> right crop, plan investment and fertilizer, check weather and soil, read
> mandi prices, detect plant disease, explore government schemes like PM-KISAN
> and KCC, and ask any farming question in Tamil or English — with responses
> streamed live from the Groq LLM (llama-3.3-70b-versatile).
>
> The system contains real data for 22+ Indian states, 70+ districts with
> climate and soil profiles, 40 major crops with complete economic profiles
> (yield, investment, profit, pests), 12 central government schemes, and a
> crop-disease diagnosis database. Every tool is personalised to the farmer's
> saved profile (location, acres, budget, water source, method).
>
> Built as a zero-dependency web app (HTML + CSS + vanilla JavaScript), it
> requires no installation, no backend, no database — it runs in any browser.
> Farm profile and chat history persist via localStorage so the farmer's plan
> follows them across sessions.

## 5. LIKELY DTM QUESTIONS — PREPARED ANSWERS

**Q: Is this just a chatbot?**
A: No. The chatbot is one of 12 features. Nine of them work entirely
   offline without AI: location picker, crop explorer, cost calculator,
   fertilizer planner, crop calendar, weather analyser, market trends,
   disease detector, and schemes guide.

**Q: Where did the data come from?**
A: State climate and soil data is aggregated from ICAR and state
   agriculture department publications. Crop economics use representative
   mandi ranges from Agmarknet. Government schemes data is from the
   official scheme portals (pmkisan.gov.in, pmfby.gov.in, enam.gov.in,
   etc.) — every scheme card links to its official source.

**Q: What AI model powers the chatbot?**
A: Groq's hosted llama-3.3-70b-versatile. We chose Groq because of its
   extremely low latency — tokens stream back in under 50 ms so the
   experience feels real-time, which matters for a chatbot that rural
   farmers may use over a weak connection.

**Q: How do you handle Tamil?**
A: Three places: (1) A language toggle in the chatbot UI switches the
   system prompt to require Tamil responses, (2) the browser's
   Web Speech API uses `ta-IN` for voice input and text-to-speech,
   (3) UI strings like placeholders and greetings are translated.

**Q: What about privacy?**
A: Farm profile, location, and chat history live in the user's own
   browser via localStorage. Nothing leaves the device except the
   specific message sent to Groq when the user asks the chatbot a question.

**Q: Can this scale to millions of farmers?**
A: The front end is static HTML so it can be served from any CDN.
   Groq's API is pay-per-token, billable on usage. The entire product
   works without a database. For production, the only changes needed would
   be moving the Groq key to a backend proxy and adding authentication.

**Q: Why didn't you use React?**
A: Deliberate choice for zero-dependency simplicity. A React build would
   require Node.js, npm install, build step, bundler — none of which
   a farm extension worker needs. Vanilla JS makes the entire app one
   double-click away from running.
